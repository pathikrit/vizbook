prefuse is a user interface toolkit for constructing highly-interactive visualizations of graph-structured (i.e. node-link) data. prefuse applications are built by stringing together composable modules (called {@link prefuse.action.Action Actions}) into {@link prefuse.activity.ActionList ActionLists} that perform various tasks such as graph processing and animation.

Other features of the toolkit include:
  • Libraries of composable actions, including layout algorithms (see the {@link prefuse.action action} and {@link prefusex.layout layout} packages).
  • Filtering and caching schemes to support large data sets through a centralized {@link prefuse.ItemRegistry ItemRegistry} .
  • A physics simulation engine supporting physically-based layout and interaction (see the {@link prefusex.force force} package).
  • Flexible activity scheduling and animation support (see the {@link prefuse.activity activity} package);
  • User interface event handling (see the {@link prefuse.event event} and {@link prefusex.controls controls} packages).
  • View transformations, as well as panning and zooming (see the {@link prefuse.Display Display} class and the {@link prefusex.distortion distortion} package).

The general pattern of building a prefuse visualization component is to first determine the {@link prefuse.graph.Graph Graph} data structure to visualize, and construct an {@link prefuse.ItemRegistry ItemRegistry} to create and store visual representations of the graph data along with one or more {@link prefuse.Display Display} instances to draw these visual representations to the screen. One then assigns a {@link prefuse.render.RendererFactory RendererFactory} to the {@link prefuse.ItemRegistry ItemRegistry}, which determines which {@link prefuse.render.Renderer Renderer} instances are used to draw individual items. Then any number of {@link prefuse.activity.ActionList ActionLists} can be created and scheduled to perform graph operations such as layout, color assignment, and animation. Finally, interactive behaviors can be specified by adding custom {@link prefuse.event.ControlListener ControlListeners} or pre-built {@link prefusex.controls Controls} to the {@link prefuse.Display Display} instances. For working examples of this process, take a look at the provided {@link prefuse.demos demonstration applications}.
